    Introduction: This document is owned and approved by **Document Owner** as a component of the company compliance regime.

    Purpose and Scope: This document details procedures and responsibilities applicable to all **Organisation** staff, associates and contractors provided with **Organisation** user accounts, who are described as “Users” within this document.

    Document Amendment: If, during the course of your duties, you become aware of any amendments that should be made to this document, you should inform the **Document Owner**.

    Introduction

    **Organisation** needs to operate in a manner which demonstrates that physical, technical and personnel risks are adequately controlled. This allows clients to trust **Organisation** to deliver engagements which involve handing client data and ensures that **Organisation** meets data protection and other legislative requirements. Users will be required to acknowledge this document before they are issued with a device or user account. All users will be required to repeat the acknowledgement on an annual basis or when there is a change to the document which is material to the user responsibilities.

    Client Data

    The following rules must be followed at all times, if you believe that you need to break any of these rules to deliver a client or business requirement you must escalate this to **Document Owner**.

    • Client data must only be stored on organisation approved laptops or organisation provided online storage
      • All organistion approved End User Devices(EUDs) must comply with the Cyber Essentials technical requirements, which includes at rest encryption and user authentication.
      • All organisation approved online storage will be located in the UK and use unique user identities and multi-factor authentication.
    • Client data must be stored in engagement specific areas of organisation provided online storage, so access is restricted to those users with a valid need to know and automatically backed up.
    • Any requests to add users to engagement areas of online storage must be approved by the Director in Charge for the engagement.
    • Client data will only be transmitted using encrypted mechanisms, only these mechanisms are available for organisation provided online storage. The use of email needs to be agreed in writing with the client or approved by **Document Owner**.
    • Client data must not be stored or exchanged on removable media.
    • If necessary, offline backups which include client data must be robustly encrypted and kept in company approved physically secure storage.
    • Paper copies of client data should not be created unless you can store them securely or destroy them at the end of the working day.
    • Paper documents must be destroyed using cross-cut shredders.
    • Client data is provided for specific engagements so you must not use it for another engagement with the same client, without first seeking advice from **Document Owner**.
    • Client data held electronically should never normally be requested by clients, because they already hold the data. If you receive such a request this must be referred to **Document Owner** prior to responding to the requestor.
    • Any requests to release client data (including Subject Access Requests, Freedom of Information Act and Legislative Requests) held by **Organisation** to any party must be referred to the **Document Owner** prior to responding to the requestor or notifying the client.
